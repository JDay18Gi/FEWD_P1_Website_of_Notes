Lesson 1 The Basics of the Web and HTML


Web Basics

  • A collection of web pages
  • Computer makes requests to servers (via HTTP). Server responds with files.
  • Web invented in 1990
  • 30+ billion pages!
  • HTML = Hypertext Markup Language. It is the base for nearly all web pages. HTML docs are the majority of documents on the web.


HTML

  • Written like below, where 'name' is a tag. (But some tags do not need to be closed.)

<name> contents </name>

  • Some examples of tags are these:

<em> contents </em>

- for italics

<b> contents </b>

- for bold

<a href="http://www.example.com/"> Example Link </a>

- for Example Link

  • In web development, an "element" refers to the opening and closing tag along with the contents of the tag.

Inline and Block Elements

  • Inline elements have no box around them (think text without a textbox). Examples of inline elements are these: <a>, <span>, <br>, and <img>, <strong>

  • Block Elements have an invisible box around them (like a textbox).

Computers are Dumb

  • We say this with compassion, but computers need all these special tags and elements because they cannot understand things the way humans do. This is important for programmers to keep in mind because even little errors can cause big problems.


Lesson 2: Structure and HTML

All Webpages are Boxes

  • These boxes make up the layout of the webpage.
  • Even circular items have invisible boxes around them!
  • Each box has a few aspects to it: margin, border, padding, and the item itself.

DOM (Document Object Model)

  • DOM refers to the tree-like structure used to write code for a webpage.
  • Elements that are most subodinate (furthest to the right) are affected by the "parent" elements which are less indented.
  • Spans and Divs are used to group code. Span is inline. Div is block (remember that means it has an imaginary box around it).

Images

  • Images can be included by using the following code:
            <img src="image_url" alt="alt_text">
  • Alt text is important for the visually impaired and if the image doesn't load.

Lesson 3: Adding CSS Style

  • Avoid repetition in your code!!!

Adding CSS Style to HTML

  • CSS = cascading style sheets
  • Cascading = the most specific rule gets applied and rules and inherited from elements further to the left.
  • inheritance: text related properties are inehrited (from the left to the elements further to the right), box related properties are not inherited.
  • CSS ID Selectors: '.', '#', etc.

Box Sizing & Structure

  • use percentages
  • use "border-box"
